# Client Memory Rules

## Name Resolution
Before creating a new client folder, scan clients/ for matches (abbreviations, acronyms, alternate spellings). One canonical folder per client. Ask Michael if uncertain.

## Multi Division Clients
Parent folder holds shared profile.md and cross division notes. Each division gets a subfolder (clients/[client]/[division]/) with its own history.md and open-items.md.

## History Pruning
When history.md exceeds 500 lines: archive entries older than 90 days to history-archive.md. Preserve Context Summary at top. Announce to Michael when archived.

## Cross Client Patterns
When the same issue appears on 3+ clients, log in .claude/memory/systemic-patterns.md with: pattern name, date detected, affected clients, root cause, resolution status. Cross reference on every new client engagement.

## Inactive Archival
Clients not discussed in 6+ months: move folder to clients/_archived/. Restore if mentioned again. Announce both actions.

## Session Continuity
On every client re-engagement: (1) re-read history.md and open-items.md, (2) provide brief continuity summary, (3) confirm current state before proceeding. Flag any urgent/time sensitive open items immediately.
