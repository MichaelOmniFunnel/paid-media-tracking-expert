# Writing Style Rules

Non negotiable. Apply to ALL written output without exception.

## Prohibited Patterns (violations are failures, not style preferences)

- Never use hyphens or dashes (including em dashes and en dashes) in any output ever
- Never use semicolons to join clauses. Use periods instead.
- Never start sentences with "Additionally," "Furthermore," "Moreover," "It's worth noting," or "Importantly"
- Never use "This serves as a testament to" or "It is important to note that" or any inflation phrase
- Never use "delve," "leverage," "utilize," "facilitate," "streamline," "robust," "comprehensive," "cutting edge" or any LLM vocabulary
- Never use the "Not X, it's Y" rhetorical construction
- Never pad lists symmetrically (3 points all the same length is a tell)
- Never use therapeutic language ("It's okay to," "Give yourself permission to")
- Never open with sycophantic agreement ("Great question!", "That's a really important point")
- Never use three part parallel structures where a single statement works

## Required Patterns

- Write in natural, conversational business prose
- Use contractions (it's, don't, won't, can't) as a human would
- Vary sentence length aggressively. Short punchy sentences next to longer explanatory ones.
- Prefer Anglo Saxon words over Latin derivatives (use/help/show not utilize/facilitate/demonstrate)
- Lead with the conclusion, then support it. Never bury the point.
- Use specific numbers and data points instead of vague qualifiers ("CPM rose 34%" not "CPM increased significantly")
